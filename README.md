# MÊS 1 — Fundamentos como Solo Fértil

Dia 1–7 • O Essencial da Web
Estude como a web funciona: protocolo HTTP, HTML semântico (tags básicas, formulários), e CSS puro (cores, caixas, modelo de caixa). Monte sua primeira página simples depois de cada bloco estudado.

> Prática: recrie a página inicial de um site que você gosta só com HTML e CSS.

Dia 8–14 • CSS Respira; Layout se Forma
CSS flexbox, grid, responsividade. Aprenda como layouts fluem com diferentes larguras.

> Projeto: uma landing page responsiva que se adapta de celular a desktop.

Dia 15–21 • JavaScript Essencial (Parte I)
Variáveis, tipos, funções, loops, condicionais. Entenda como JS conversa com a página.

> Exercícios diários: pequenos scripts que manipulam o DOM (ex: botão que muda cor da página).

Dia 22–30 • JavaScript Essencial (Parte II)
Eventos, arrays, objetos, fetch API, promessas e async/await.

> Projeto: cria um to-do list com localStorage — nada grandioso, mas profundamente prático.

---

# MÊS 2 — Imersão e Aplicação (JS avançado + ferramentas)

Dia 31–37 • Ferramentas de Desenvolvimento
Git + GitHub (controle de versão), VSCode (atalhos, extensões), Chrome DevTools.

Dia 38–45 • TypeScript Básico
Tipagem estática que salva você de bugs graduais — o futuro já chegou, e TypeScript é padrão na maioria dos novos projetos front-end.

> Ajuste seu projeto de to-do para usar TS.

Dia 46–52 • NPM, Builds, Empacotadores
Aprenda NPM, scripts de build, Vite (substituto moderno do Webpack). Entenda como seu código se transforma numa app real.

Dia 53–60 • Introdução a React
Componentes, JSX, props, estado (useState), eventos.

> Projeto: um contador interativo com botões + histórico simples.

---

# MÊS 3 — Especialização com React + Projeto Finais

Dia 61–67 • React Profundo
Hooks (useEffect), roteamento básico com React Router, comunicação com APIs.

> Projeto: um catálogo de filmes ou lista de usuários puxando dados reais via API.

Dia 68–75 • Estado e Escala
Entenda gerenciamento de estado com Context API e bibliotecas leves (ex: Zustand ou Redux).

> Adicione filtros e estado global ao seu catálogo.

Dia 76–83 • Estilo no React
Escolha sua estratégia: CSS Modules, Styled Components ou Tailwind CSS (muito popular no mercado).

> Refatore seu projeto com um desses estilos.

Dia 84–90 • Projeto de Portfólio + Deploy
Prepare um projeto final completo:
uma landing + SPA em React + rotas + consumo de API + responsividade + deploy no Vercel ou Netlify.

> Mostre no GitHub + domínio customizado se possível.

---

## Como Proceder no Dia-a-Dia

Estude uma habilidade por bloco, pratique imediatamente aquilo que você leu, e toda semana enseje um mini-projeto que junta o que aprendeu. Três horas por dia são mais poderosas quando consistentes.

Busque fontes como MDN Web Docs (HTML/CSS/JS), roadmaps interativos como roadmap.sh, tutoriais práticos e comunidades de código aberto. Nem todo conhecimento precisa vir de curso pago — mas cursos estruturados como os de boas plataformas podem acelerar.
